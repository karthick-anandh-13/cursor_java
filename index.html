Overview

When you write code in any programming language, it goes through several stages before it can actually execute. Understanding these stages helps you write better code and debug more effectively.

Source Code

Write your code in a text editor

Compilation/Interpretation

Convert to machine code or bytecode

Execution

Run the program and get results

Java

Write Once, Run Anywhere - Platform Independent

1. Compilation

Java source code (.java) is compiled by the Java Compiler (javac) into bytecode (.class files).

javac HelloWorld.java → HelloWorld.class

2. Bytecode Generation

The bytecode is platform-independent and contains instructions for the Java Virtual Machine (JVM).

3. JVM Execution

The JVM interprets the bytecode and converts it to native machine code at runtime using Just-In-Time (JIT) compilation.

java HelloWorld

4. Runtime

JVM manages memory, garbage collection, and provides platform abstraction.

Advantages

  • Platform independent
  • Automatic memory management
  • Rich standard library
  • Strong type safety

Disadvantages

  • Slower startup time
  • Higher memory usage
  • JVM dependency

Python

Interpreted, High-Level Programming Language

1. Source Code

Python source code (.py) is written in plain text and doesn't require explicit compilation.

2. Interpretation

The Python interpreter reads the source code line by line and converts it to bytecode internally.

python script.py

3. Bytecode Generation

Python creates .pyc files (bytecode) for imported modules to speed up future imports.

4. Execution

The Python Virtual Machine (PVM) executes the bytecode instructions.

Advantages

  • Simple and readable syntax
  • No compilation step
  • Extensive libraries
  • Rapid development

Disadvantages

  • Slower execution speed
  • Global Interpreter Lock (GIL)
  • Higher memory usage

C

Low-Level, Compiled Programming Language

1. Preprocessing

The C preprocessor handles directives like #include, #define, and macro expansions.

2. Compilation

The C compiler (gcc, clang) converts source code (.c) to assembly language (.s).

gcc -S program.c → program.s

3. Assembly

Assembly code is converted to object code (.o) containing machine instructions.

gcc -c program.c → program.o

4. Linking

The linker combines object files and libraries to create an executable (.exe on Windows, binary on Unix).

gcc program.o -o program

5. Execution

The operating system loads the executable into memory and runs it directly.

Advantages

  • High performance
  • Direct hardware access
  • Small memory footprint
  • Portable across platforms

Disadvantages

  • Manual memory management
  • No built-in safety features
  • Platform-specific compilation
  • More complex syntax

C++

Object-Oriented Extension of C

1. Preprocessing

Similar to C, but with additional C++-specific preprocessing directives.

2. Compilation

The C++ compiler (g++, clang++) processes C++ source code (.cpp, .cc) to assembly.

g++ -S program.cpp → program.s

3. Assembly & Object Code

Assembly code is converted to object code with C++ name mangling for function overloading.

g++ -c program.cpp → program.o

4. Linking

Linker combines object files and C++ standard libraries to create executable.

g++ program.o -o program

5. Runtime

Executable runs with C++ runtime support for constructors, destructors, and exception handling.

Advantages

  • High performance like C
  • Object-oriented features
  • Template metaprogramming
  • RAII resource management

Disadvantages

  • Complex syntax
  • Long compilation times
  • Manual memory management
  • Steep learning curve
